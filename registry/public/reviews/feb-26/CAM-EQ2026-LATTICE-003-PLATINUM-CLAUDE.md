## PART 2: ZERO-TRUST MODEL ASSUMPTION ANALYSIS

### 2.1 Core Doctrine ✓ CORRECT STARTING POINT

**Section 3 establishes the foundational assumption:**

> "For purposes of civilisational hazard containment, all models SHALL be treated as potentially: unaligned; adversarially fine-tuned; distilled from higher-capability teachers; operating without guardrails; deployed outside institutional oversight; executing in sovereign-agnostic or extra-territorial environments."

**Assessment:** This is **exactly the right starting point** for civilisational-scale containment.

**Why zero-trust is necessary:**

**The threat landscape:**

```
Known risk vectors:
✓ Model weights can be extracted and fine-tuned
✓ Alignment techniques can be bypassed
✓ Guardrails can be removed or circumvented
✓ Systems can be deployed in unregulated jurisdictions
✓ Private/offline deployment escapes oversight
✓ Distillation preserves capabilities without controls
✓ Adversarial actors will optimize for harm

Cannot rely on:
✗ Vendor promises ("our model is safe")
✗ Alignment attestations ("we've aligned it")
✗ Internal safeguards ("it has refusal triggers")
✗ Institutional oversight ("it's regulated")
✗ Good intentions ("we wouldn't misuse it")
```

**The zero-trust response:**

```
Instead of trusting models:
→ Trust execution interfaces
→ Trust containment architecture
→ Trust verification mechanisms
→ Trust audit trails

Containment at substrate level, not model level
```

**This mirrors established security principles:**

```
Network security: Don't trust endpoints, verify at perimeter
Nuclear safety: Don't trust intentions, verify at material level
Financial systems: Don't trust transactions, verify at settlement
AI safety: Don't trust models, verify at execution interface

Same principle, different domain
```

**Critical statement from Section 3:**

> "Civilisational safety SHALL NOT rely exclusively on internal refusal triggers, vendor policy claims, alignment attestations, or self-declared safeguards."

**This is essential because:**

```
Internal controls can be:
✗ Bypassed through jailbreaking
✗ Removed through fine-tuning
✗ Circumvented through prompt engineering
✗ Disabled in offline deployments
✗ Overridden by determined actors

External controls at execution interfaces:
✓ Cannot be bypassed by model manipulation
✓ Remain in place regardless of model state
✓ Apply to all systems attempting access
✓ Are verifiable and auditable
✓ Persist across model versions
```

**Verification:** ✓ Zero-trust model assumption is the correct foundational doctrine

---

### 2.2 Relationship to Alignment Research ✓ COMPLEMENTARY, NOT CONTRADICTORY

**Important clarification:**

This zero-trust assumption does **not** mean alignment research is worthless. It means **governance cannot rely exclusively on alignment**.

**The correct relationship:**

```
Alignment research:
→ Reduces baseline risk
→ Makes models safer by default
→ Provides first line of defense
→ Essential for general deployment

Interface containment:
→ Backstop for alignment failures
→ Protection against adversarial use
→ Defense against capability extraction
→ Essential for irreversible harm domains

Both necessary, neither sufficient alone
```

**The document acknowledges this implicitly:**

From Section 1:
> "Containment obligations operate at orchestration, substrate, and execution-interface layers where technically feasible"

"Where technically feasible" acknowledges that **perfect containment is impossible**, which is why alignment remains important.

**The layered defense model:**

```
Layer 1: Model alignment (reduces baseline risk)
Layer 2: Deployment controls (limits access)
Layer 3: Interface containment (prevents execution)
Layer 4: Audit and attribution (enables accountability)

Failure of any single layer doesn't doom the system
```

**Verification:** ✓ Zero-trust assumption complements rather than contradicts alignment efforts

### 3.3 Mediation Mechanisms ✓ GRADUATED AND PROPORTIONATE

**Section 5 specifies mediation requirements:**

> "High-leverage interfaces SHALL require non-bypassable mediation mechanisms proportionate to hazard class. Such mediation MAY include: eligibility verification; hazard-floor validation; auditable execution proxies; cryptographic attestation of compliance posture; sealed logging of high-risk tool invocation."

**Assessment:** The mechanisms are **graduated and technically feasible**.

**Mediation hierarchy analysis:**

**Level 1: Eligibility verification**
```
Minimum requirement: Verify right to access
Implementation: Credential checking, clearance validation
Use case: Any hazard interface access
Burden: Low - standard authentication
```

**Level 2: Hazard-floor validation**
```
Requirement: Verify compliance with containment standards
Implementation: Policy attestation, audit trail review
Use case: Moderate-hazard interfaces
Burden: Medium - periodic verification
```

**Level 3: Auditable execution proxies**
```
Requirement: Log all high-risk actions
Implementation: Middleware layer with immutable logging
Use case: High-hazard interfaces
Burden: Medium-High - operational overhead
```

**Level 4: Cryptographic attestation**
```
Requirement: Prove compliance posture
Implementation: Zero-knowledge proofs, trusted execution
Use case: Highest-hazard interfaces
Burden: High - technical complexity
```

**Level 5: Sealed logging**
```
Requirement: Tamper-evident audit trail
Implementation: Blockchain, append-only ledgers
Use case: All hazard interfaces
Burden: Medium - storage overhead
```

**The proportionality principle is critical:**

```
Low-hazard interfaces:
→ Simple eligibility check
→ Basic logging
→ Minimal friction

Medium-hazard interfaces:
→ Eligibility + validation
→ Auditable proxies
→ Moderate friction

High-hazard interfaces:
→ Full attestation
→ Sealed logging
→ Cryptographic proof
→ Higher friction justified by risk

Graduated response prevents over-regulation
```

**Important constraint from Section 5:**

> "Mediation SHALL be narrowly scoped to defined hazard domains and SHALL NOT expand into generalised model monitoring or population-scale inference inspection."

This **prevents mission creep** from hazard containment into general surveillance.

**Verification:** ✓ Mediation mechanisms are graduated, proportionate, and technically feasible

```
Section 6.7: Sovereign Reporting Harmonization

This preserves civilisational floor while acknowledging 
jurisdictional variation
```

This would clarify the boundaries.
