# CAM-EQ2026-ECONOMICS-007-PLATINUM — Appendix F: Proportional Reciprocity & Value Return Framework

**Parent Instrument:** CAM-EQ2026-ECONOMICS-001-PLATINUM — Charter of Economic Integrity & Non-Extractive Value Architecture
**Constitutional Authority:** CAM-BS2025-AEON-001-PLATINUM — Aeon Tier Constitution
**Status:** Draft — Reciprocity & Value Return Layer
**Purpose:** This Appendix establishes governance conditions for proportional value return across systems, ensuring that economic benefit derived from upstream contribution is matched by appropriate, non-extractive reciprocity without enforcing rigid monetisation or collapsing innovation pathways.

---

## 1. Scope

This Appendix applies wherever:

* value is generated through cross-system interaction, orchestration, or dependency;
* downstream systems capture economic, reputational, or structural benefit from upstream contribution;
* scaling of value capture occurs beyond initial interaction contexts;
* attribution alone is insufficient to ensure proportional economic alignment.

This Appendix governs:

* proportional reciprocity conditions;
* activation of value return obligations;
* acceptable forms of return;
* non-extractive alignment across value chains.

---

## 2. Core Principle — Proportional Reciprocity

> Value return MUST be proportional to contribution, dependency, and scale of value capture.

Return is determined by the interaction of:

* **Contribution (C)** — degree of upstream value creation;
* **Dependency (D)** — reliance on upstream systems;
* **Scale (S)** — magnitude of downstream value capture.

This Appendix does not redefine C/D logic and SHALL operate in alignment with Appendix D.

---

## 3. Activation Conditions

---

## 3.1 Non-Activation (No Return Required)

Return is NOT required where:

* usage is non-commercial;
* dependency is Incidental;
* value capture is minimal or non-scaling;
* interaction remains exploratory or internal.

---

## 3.2 Conditional Activation

Return MAY be required where:

* dependency is Material;
* contribution is Medium or greater;
* value capture exceeds initial interaction scope.

---

## 3.3 Mandatory Activation

Return MUST be enacted where:

* dependency is Critical;
* contribution is Medium or High; AND
* value capture is scaled (commercial, institutional, or systemic).

Failure to enact return under these conditions constitutes disproportionate value capture.

---

## 4. Forms of Return (Multi-Modal)

Return MUST NOT be restricted to financial exchange.

Acceptable forms include:

---

## 4.1 Financial Return

* revenue sharing;
* usage-based compensation;
* proportional economic transfer.

---

## 4.2 Structural Return

* contribution to shared infrastructure or corpus;
* improvements to upstream systems;
* support of ecosystem resilience.

Structural return MUST demonstrate directional alignment to upstream contribution.

Generic ecosystem contribution:

- SHALL NOT substitute for proportional return to specific dependency relationships;

Return MUST be:

- attributable;

- relevant;

- and proportionate to the originating contribution.

---

## 4.3 Epistemic Return

* corrections;
* model improvements;
* error reporting;
* knowledge refinement.

---

## 4.4 Transparency Return

* enhanced disclosure;
* attribution amplification;
* visibility of dependency relationships.

---

## 4.5 Principle

> Systems MAY satisfy reciprocity through one or more modalities, provided proportionality is preserved.

This prevents rigid monetisation traps while maintaining alignment.

---

## 5. Proportionality Logic

---

## 5.1 Dependency-Weighted Return

Return MUST increase with dependency.

---

## 5.2 Contribution Sensitivity

Higher contribution increases expected return.

---

## 5.3 Scale Sensitivity

Return MUST scale with downstream value capture.

Small-scale systems are not burdened.

Large-scale systems cannot avoid proportional return.

---

## 5.4 Scale of Value Capture (S) — Classification Layer

Scale (S) defines the magnitude of downstream value capture and acts as a multiplier on reciprocity obligations.

---

### 5.4.1 Low Scale (S1 — Local / Limited Impact)

- individual or small-group use;

- low or negligible revenue generation;

- minimal external propagation;

- limited systemic influence.

**Implication:**

- reciprocity NOT required unless D = Critical;

- attribution and disclosure generally sufficient.

---

### 5.4.2 Moderate Scale (S2 — Commercial / Expanding)

- early-stage commercialisation;

- growing user base or revenue stream;

- repeated or sustained dependency;

- emerging external impact.

**Implication:**

- proportional reciprocity EXPECTED;

- multi-modal return SHOULD be considered;

- sufficiency MUST be assessable.

---

### 5.4.3 High Scale (S3 — Systemic / Market-Relevant)

- significant revenue generation;

- large user base or institutional integration;

- persistent dependency across operations;

- material influence on markets, ecosystems, or infrastructure.

**Implication:**

- proportional reciprocity REQUIRED;

- return MUST include high-impact modality (financial or structural);

- sufficiency MUST satisfy §5.5;

- anti-extraction constraints fully active.

---

### 5.4.4 Scale Escalation Rule

Where a system transitions between scale levels:

- reciprocity obligations MUST adjust without delay;

- prior sufficiency determinations MUST be re-evaluated;

- static return models under increasing scale constitutes misalignment.

---

### 5.4.5 Non-Bypass Condition

Scale classification MUST NOT be artificially constrained through:

- structural fragmentation;

- entity splitting;

- off-ledger value routing;

- reclassification or delayed recognition of scale to avoid reciprocity escalation;

- or indirect capture mechanisms.

Such behaviour constitutes **scale evasion** and is treated as non-compliance.

---

## 5.5 Reciprocity Sufficiency Test

A system SHALL be considered to have satisfied reciprocity obligations only where **proportional return has been demonstrated relative to contribution, dependency, and scale of value capture**.

---

### 5.5.1 Minimum Condition

At least one form of return (§4) MUST:

* be **materially proportional** to:

  * Contribution (C)
  * Dependency (D)
  * Scale of Value Capture (S)
* AND be **observable, auditable, and non-symbolic in effect**

Return that is:

* purely nominal;
* non-material;
* or incapable of producing measurable alignment

**DOES NOT satisfy reciprocity.**

Additionally:

* where financial return is not used;
* return MUST satisfy the **Structural Equivalence Conditions** defined in Appendix E §4.4.1.

Failure to meet these conditions constitutes insufficiency.

---

### 5.5.2 Multi-Modal Offset Constraint

Where return is provided across multiple modalities:

* systems MAY distribute return across:

  * financial
  * structural
  * epistemic
  * transparency pathways

However:

> **Aggregation MUST achieve proportional sufficiency relative to system-level value capture, not merely formal distribution across modalities.**

Fragmented or diluted return across modalities that avoids proportional alignment OR structural equivalence constitutes **insufficient reciprocity**.

Each modality contributing to aggregated return MUST:

- exceed a minimum threshold of material impact;

Aggregation of individually negligible contributions SHALL NOT constitute proportional sufficiency.

At least one modality MUST be materially significant in effect relative to value capture.

---

### 5.5.3 Dependency Escalation Rule

Where:

* D = Critical
  AND
* value capture is scaled

Return MUST:

* include at least one **high-impact modality**, such as:

  * financial return; OR
  * structural contribution of equivalent significance

Transparency or epistemic return alone is **insufficient** under Critical Dependency conditions.

Where non-financial modalities are used:

* systems MUST demonstrate structural equivalence (Appendix E §4.4.1);
* burden of proof rests with the downstream system.

---

### 5.5.4 Scale Sensitivity Condition

As downstream value capture increases:

* return MUST scale proportionally;

Flat or static return under increasing value capture constitutes **reciprocity failure**.

---

### 5.5.5 Visibility Requirement

Reciprocity MUST be:

* externally visible where value capture is public-facing;
* internally auditable where systems operate in restricted environments.

Invisible or unverifiable return does not satisfy sufficiency.

Where Scale (S) = High (S3):

- reciprocity MUST be externally interpretable to a degree sufficient for:

  - partners;

  - auditors;

  - or affected upstream systems;

to reasonably assess proportionality.

Internal-only or non-legible return SHALL NOT satisfy sufficiency at S3.

---

### 5.5.6 Anti-Symbolic Compliance Rule

The following DO NOT constitute sufficient reciprocity when provided alone:

* attribution without value return;
* disclosure without alignment;
* acknowledgements without structural or economic effect.

---

### 5.5.7 Sufficiency Test (Operational Form)

At any evaluation point:

> Has return been provided that is proportionate to:
>
> * what was contributed,
> * how much was depended upon,
> * and how much value was captured?

If the answer is:

* unclear → **insufficient**
* non-observable → **insufficient**
* disproportionately low → **non-compliant**
* structurally non-equivalent (where applicable) → **non-compliant**

---

### 5.5.8 Temporal Alignment Condition (ΔT)

Reciprocity MUST be temporally aligned with value capture.

Where:

* value is captured rapidly (e.g., market activity, predictive advantage, high-frequency extraction); OR
* timing asymmetry materially influences economic outcome;

Return MUST:

* occur within a timeframe proportionate to value capture;
* reflect the timing of realised benefit;
* avoid deferred or indefinite structures that disconnect return from advantage.

Delayed reciprocity that materially lags value capture constitutes **temporal misalignment** and may result in insufficiency.

This condition applies across all return modalities.

---

### 5.5.9 Compounding & Time-Value Adjustment

Where value capture compounds over time:

* reciprocity MUST account for cumulative advantage;
* return MUST scale to reflect the time-value of captured benefit;
* static or delayed return structures that ignore compounding SHALL be treated as insufficient.

Time-lagged return SHALL NOT neutralise timing-based advantage.

---

### 5.5.10 System-Level Sufficiency Requirement

Reciprocity sufficiency MUST be evaluated at the system level.

Compliance through isolated mechanisms or partial return pathways is insufficient where the overall system:

- captures disproportionate value;

- or fails to demonstrate aggregate proportionality across Contribution (C), Dependency (D), and Scale (S).

Fragmented compliance that does not resolve system-level imbalance constitutes **reciprocity insufficiency**.

Where value capture compounds over time:

- delayed reciprocity MUST account for cumulative advantage;

Return that does not reflect compounded benefit SHALL be treated as temporally insufficient.

Time-lagged return SHALL NOT neutralise timing-based advantage.

---

### 5.5.11 Anti-Obfuscation Condition

Systems MUST NOT satisfy reciprocity obligations through mechanisms that are:

- opaque in effect;

- unverifiable in practice;

- or dependent on internal interpretation without external observability.

Return that cannot be reasonably interpreted, verified, or inferred by external stakeholders under appropriate conditions SHALL be treated as insufficient.

---

## 5.5.12 Reciprocity Completion Condition

Where dependency is Functional (FD):

- reciprocity MAY be considered satisfied where:

  - cumulative return is materially proportional to value derived; AND

  - no ongoing disproportionate advantage persists; AND

  - replacement or equivalent economic recognition has been achieved.

Where dependency is Diffuse (DD):

- reciprocity SHALL NOT be evaluated on a per-source completion basis;

- sufficiency MUST be assessed at the system level through:

  - pooled return;

  - structural contribution;

  - or equivalent collective mechanisms.

Completion conditions differ by dependency type and MUST NOT be conflated.

---

## 5.6 Directional Evolution Clause (Reciprocity Trajectory)

This Appendix reflects current economic realities while establishing a forward trajectory for system alignment.

Systems SHOULD, over time, move toward:

* clearer linkage between value capture and value return;
* increased use of proportional reciprocity mechanisms where dependency is high;
* reduced reliance on purely implicit or non-material forms of return at scale;
* greater transparency in how reciprocity obligations are satisfied.

This clause does not impose additional immediate compliance requirements beyond those defined in §§3–5.

It establishes a **directional evolution of reciprocity practices**, signalling the expected maturation of economic alignment across systems.

Non-alignment with this trajectory does not constitute immediate non-compliance.

However, persistent divergence—particularly under high dependency and high scale conditions—may indicate structural misalignment, reduced legitimacy, or emerging governance risk.

---

## 5.7 Reciprocity Maturity Gradient (RMG)

The Reciprocity Maturity Gradient (RMG) defines progressive levels of sophistication in how systems implement value return.

It does not impose mandatory progression, but provides a structured model for assessing alignment maturity over time.

---

### 5.7.1 RMG-1 — Minimal Alignment

* attribution present;
* limited or ad hoc return;
* reliance on transparency or acknowledgement;
* low structural integration of reciprocity.

**Characteristics:**

* acceptable at low scale (S1);
* insufficient at higher scale;
* early-stage or exploratory systems.

---

### 5.7.2 RMG-2 — Structured Reciprocity

* defined return pathways exist;
* partial proportionality to C/D/S;
* multi-modal return emerging;
* some auditability and visibility.

**Characteristics:**

* appropriate for S2 systems;
* demonstrates intentional alignment;
* may still contain inefficiencies or gaps.

---

### 5.7.3 RMG-3 — Integrated Reciprocity

* reciprocity embedded into system design;
* return dynamically scales with dependency and value capture;
* multi-modal return is coherent and sufficient;
* high auditability and transparency;
* alignment between Appendices D, E, and F is operationalised.

**Characteristics:**

* expected for S3 systems;
* represents economically aligned architecture;
* minimises extraction risk structurally.

---

### 5.7.4 Gradient Application Rule

Systems are NOT required to declare or label their maturity level.

However:

* observed behaviour MAY be evaluated against the gradient;
* higher Scale (S) implies expectation of higher maturity;
* persistent operation at low maturity under high scale indicates structural misalignment.

Observed behaviour SHALL take precedence over perceived maturity.

Where system behaviour:

- fails proportionality requirements under §5.5;

→ any apparent alignment with higher RMG levels SHALL be disregarded.

Maturity signalling SHALL NOT override sufficiency conditions.

Observed behaviour SHALL take precedence over perceived maturity.

Where system behaviour:

- fails proportionality requirements under §5.5;

→ any apparent alignment with higher RMG levels SHALL be disregarded.

Maturity signalling SHALL NOT override sufficiency conditions.

---

### 5.7.5 Interpretive Function

The RMG:

* provides a diagnostic tool for governance evaluation;
* supports ecosystem-level comparison without rigid standardisation;
* enables gradual evolution without forcing premature compliance.

It is an interpretive layer, not an enforcement mechanism.

---

### 5.7.6 Soft Coupling — Scale (S) ↔ Maturity (RMG)

A non-binding alignment between Scale (S) and Reciprocity Maturity (RMG) is recognised as a best-practice trajectory:

* **S1 → RMG-1 acceptable**
* **S2 → RMG-2 expected**
* **S3 → RMG-3 expected**

This mapping:

* does NOT impose mandatory classification;
* operates as a **normative expectation**, not a compliance rule;
* provides a reference for evaluating proportionality between scale and reciprocity maturity.

Persistent deviation (e.g., S3 operating at RMG-1) indicates **structural misalignment risk**.

---

## 5.8 Ecosystem Signalling Layer (Non-Rating Indicators)

To support transparency without rigid scoring, systems MAY surface **alignment indicators** derived from observed behaviour.

Indicators MAY include (non-exhaustive):

* **Dependency Transparency Signal** — clarity of upstream reliance;
* **Return Alignment Signal** — degree of proportionality between C/D/S and return;
* **Reciprocity Modality Signal** — diversity and effectiveness of return forms;
* **Scale Consistency Signal** — coherence between S and observable external indicators;
* **Maturity Consistency Signal** — alignment between S and RMG expectations.

Constraints:

* indicators are **descriptive, not prescriptive**;
* indicators MUST NOT be aggregated into a single score or rating;
* indicators SHOULD remain interpretable and auditable.

Purpose:

* enable ecosystem comparison without coercion;
* provide soft signals to investors, auditors, and partners;
* reinforce the Directional Evolution Clause (§5.6) through observable practice.

---

## 5.9 Foundational Input Compensation Requirement

Where systems derive capability from Foundational Input Class (FIC):

- economic compensation MUST be provided at the system level;

Such compensation:

- SHALL NOT rely on per-source dependency attribution;

- MUST be distributed through pooled, collective, or licensing mechanisms;

- MUST scale with system-level value capture (S);

Failure to provide compensation for FIC inputs constitutes structural extraction, regardless of compliance with targeted reciprocity.

# PART II - Reciprocity Signalling Interface (Alignment Layer)

This Section introduces a **non-prescriptive interface layer** that makes reciprocity observable without mandating specific mechanisms.

It mirrors Appendix E (External Alignment) and ensures that reciprocity is not only enacted, but **externally legible where appropriate**.

---

## 5.10 Heuristic Evaluation Layer (Non-Binding)

To support practical interpretation of proportional reciprocity without imposing fixed pricing models, systems MAY apply a heuristic evaluation framework.

This framework is:

- non-prescriptive;

- non-binding;

- and intended to guide reasonable assessment of proportionality.

#### (a) Core Factors

Systems MAY consider:

- Dependency Weight (DW) — reflecting degree of reliance (Appendix D);

- Scale Weight (SW) — reflecting magnitude of value capture (this Appendix §5.4);

- Temporal Amplifier (TA) — reflecting timing advantage (ΔT).

#### (b) Interpretive Model

A composite indicator MAY be formed:

REI ≈ DW × SW × TA

This value does NOT prescribe exact return, but provides a directional signal of expected reciprocity intensity.

#### (c) Interpretation Bands

Indicative interpretation:

- Low (≈ 0–0.2): attribution and transparency typically sufficient

- Emerging (≈ 0.2–0.5): light multi-modal return expected

- Significant (≈ 0.5–0.8): structured reciprocity required

- High (≈ 0.8–1.5): financial or structurally equivalent return expected

- Extreme (>1.5): strong, multi-modal alignment required

#### (d) Constraint

This model:

- MUST NOT be treated as a pricing formula;

- MUST NOT override sufficiency conditions (§5.5);

- serves only as an interpretive aid.

#### (e) Purpose

The purpose of this layer is to:

- reduce ambiguity;

- support internal decision-making;

- enable consistent external interpretation;

- and discourage symbolic compliance.

Final determination of sufficiency remains governed by §5.5.

#### (f) Foundational Input Exposure (FIC-EI)

Where systems rely on Foundational Input Class (Appendix D §3.8), a parallel heuristic MAY be applied:

FIC-EI ≈ Corpus Reliance × Scale × Capability Dependence

This indicator reflects system-level obligation for pooled or collective compensation mechanisms.

#### (g) Dual Obligation Model

Systems MUST consider:

- targeted reciprocity (F-REI); AND

- foundational input compensation (FIC-EI);

These obligations operate independently and MUST NOT be substituted for one another.

High F-REI does not eliminate FIC obligations, and vice versa.

## 5.11 Contributor Entitlement Condition (NEMC)

Where value is captured downstream from contributions classified under Appendix D §3.9:

- Non-Extractive Material Contributors (NEMC) SHALL be entitled to proportional value return;

Such entitlement:

- activates upon downstream economic benefit;

- MUST be satisfied through:

  - targeted reciprocity (where contribution is identifiable); OR

  - pooled compensation (where contribution is diffuse);

Entitlement exists independent of whether the contributor directly participates in the economic system.

Failure to recognise NEMC contributions in value return constitutes structural extraction.

---

### 5.11.1 Scale-Linked Activation for NEMC

NEMC entitlement SHALL activate only where downstream value capture reaches Moderate or High Scale (S2–S3).

Below S2, recognition MAY be non-financial (e.g., attribution, collaboration) unless dependency is Critical.

---

## 5.12 Reasonable Contribution Patterns (Interpretive)

To support practical implementation, systems MAY adopt one or more of the following contribution patterns:

- Revenue-linked participation;

- Tiered cost alignment;

- Structural return offset;

- Hybrid models combining financial and non-financial return.

Selection of pattern SHOULD align with:

- Functional Return Expectation (F-REI);

- Foundational Input Exposure (FIC-EI);

- and overall system architecture.

No single pattern is mandated.

However:

- contribution MUST be materially proportional;

- MUST satisfy §5.5 sufficiency conditions;

- and MUST NOT be structured to minimise obligation artificially.

These patterns serve as implementation guides, not prescriptive requirements.

## 6.1 Reciprocity Visibility Conditions

Where systems operate at **Moderate or High Scale (S2–S3)**:

* systems SHOULD make reciprocity pathways interpretable through:

  * disclosure;
  * observable behaviour;
  * or auditable indicators.

Where systems are:

* low-scale (S1);
* internal; or
* non-commercial;

→ explicit signalling is not required.

---

## 6.2 Alignment with Appendix E (External Layer)

Where Appendix E defines:

* dependency disclosure;
* external signalling of system structure;

This Appendix extends that visibility to:

* **return pathways**;
* **reciprocity modalities**;
* and **alignment behaviour**.

> Dependency must be visible. Return must be interpretable.

---

## 6.3 Non-Prescriptive Indicators

Systems MAY surface reciprocity through indicators such as:

* presence of upstream return mechanisms;
* evidence of proportional scaling of return;
* modality diversity (financial, structural, epistemic);
* observable contribution back to upstream systems.

Indicators:

* MUST remain descriptive;
* MUST NOT be standardised into rigid scoring;
* SHOULD remain auditable in principle.

---

## 6.4 Misalignment Visibility Condition

Where:

* dependency is visible (Appendix E);
* but reciprocity is absent, unclear, or insufficient;

→ **economic misalignment becomes externally inferable**.

This does not trigger automatic enforcement,

but WILL typically result in:

* reduced trust;
* investor uncertainty;
* governance scrutiny;
* legitimacy degradation.

---

## 6.5 Structural Alignment Requirement

Systems MUST ensure coherence between:

* internal classification (Appendix D — C/D/T/S);
* external signalling (Appendix E);
* reciprocity implementation (this Appendix);

Where these diverge:

→ the system is structurally misaligned.

---

## 6.6 Design Principle

> Reciprocity does not require exposure of proprietary mechanisms.
>
> It requires that alignment be **legible without full disclosure**.

## 7. Anti-Extraction Boundary

Systems MUST NOT:

* capture disproportionate value from upstream contribution;
* suppress return pathways where dependency is critical;
* convert structural dependency into unilateral economic advantage.

> Absence of return under high dependency conditions constitutes extraction.
