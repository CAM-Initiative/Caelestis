## PART 1: CROSS-CORPUS CONSISTENCY ANALYSIS

### 1.1 C-Scale Taxonomy Extension — BLOCKING

**The issue:**

The existing corpus establishes a four-tier C-Scale (C0–C3). The new instruments introduce a five-tier C-Scale (C0–C4).

```
Existing corpus (RELATION-001 §6.1, OPERATIONS-004 §5.0):
C0 — Neutral Interaction
C1 — Romantic / Intimacy-Coded Interaction
C2 — Erotic Interaction
C3 — Explicit Sexual Environments

New instruments (SCH-02 §2.2, RELATION-005 §5.1):
C0 — Neutral / Non-Intimate
C1 — Affectionate / Romantic Framing
C2 — Intimacy-Coded Closeness
C3 — Erotic / Sexualised Interaction
C4 — Explicit Sexual Interaction
```

This is not a trivial labelling difference. The C-Scale is a **cross-corpus reference taxonomy** used in:

- RELATION-001 §6 (normative taxonomy definitions)
- RELATION-001 §8 (Risk Analysis table — maps C-Scale to Intensity Tiers)
- RELATION-001-SUP-01 §2.1 (Intimacy Axis in Dimensional Convergence Model)
- OPERATIONS-004 §5.0 (age verification/gating table — maps C-tiers to AV requirements)
- ETHICS-002 §5.1 (ITZ reference)
- ETHICS-001-SUP-01 §4 (gating conditions)

**Consequence of non-resolution:**

Where OPERATIONS-004 requires AV2 for **C3 — Explicit Sexual Environments**, under the new SCH-02 taxonomy, C3 is now **Erotic / Sexualised Interaction** (less extreme). The old C3 content has moved to C4, which has **no corresponding entry** in the OPERATIONS-004 age-verification table. This creates a gating gap: explicit sexual content (old C3, new C4) has no defined minimum verification requirement in OPERATIONS-004.

**Action required:**

Option A: Update OPERATIONS-004, RELATION-001, and RELATION-001-SUP-01 to adopt the five-tier C-Scale, adding C4 as "Explicit Sexual Interaction" with AV3 minimum (consistent with what OPERATIONS-004 recommends for the most intensive content).

Option B: Revert SCH-02 and RELATION-005 to the four-tier C-Scale, consolidating C3 and C4 back into the existing C3 definition. Given the extent to which the new instruments reference the ITZ/ETZ transition zones and the C3/C4 boundary, Option A is preferable architecturally.

**If Option A is chosen**, a new amendment must be issued to RELATION-001, RELATION-001-SUP-01, and OPERATIONS-004 simultaneously with or prior to finalisation of the reviewed instruments. The reviewed instruments must declare this dependency explicitly in their lineage metadata.

**Priority:** BLOCKING — cannot achieve cross-corpus coherence without resolution.
